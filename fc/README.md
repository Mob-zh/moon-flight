这个`fc`文件夹是飞控（Flight Controller）的**系统核心模块目录**，负责飞控的 “初始化、配置管理、任务调度、遥控器交互” 等基础支撑功能，是飞控启动和运行的 “中枢框架”。以下是各文件的核心作用分类：

### 一、系统初始化与基础配置

| 文件                 | 作用                                                         |
| -------------------- | ------------------------------------------------------------ |
| `init.c/h`           | 飞控启动的**总初始化入口**：负责调用硬件驱动、核心模块（如 IMU、PID）的初始化，是飞控从 “上电” 到 “就绪” 的流程控制器。 |
| `board_info.c/h`     | 板级信息管理：存储飞控硬件的型号、引脚配置等信息，实现 “硬件与软件的解耦”（适配不同飞控板）。 |
| `runtime_config.c/h` | 运行时配置管理：存储飞控的动态参数（如当前飞行模式、解锁状态），是模块间共享状态的核心载体。 |

### 二、任务与调度管理

| 文件           | 作用                                                         |
| -------------- | ------------------------------------------------------------ |
| `tasks.c/h`    | 任务管理：定义飞控的所有任务（如陀螺仪采样、PID 计算），为调度器提供任务列表和属性（优先级、周期）。 |
| `dispatch.c/h` | 任务分发：辅助调度器管理任务的启动、暂停、优先级调整，是调度器的功能扩展层。 |

### 三、遥控器与控制交互

| 文件                        | 作用                                                         |
| --------------------------- | ------------------------------------------------------------ |
| `rc.c/h`、`rc_controls.c/h` | 遥控器指令处理：接收遥控器的原始信号（如 SBUS/CRSF），转换为 “滚转、俯仰、油门、偏航” 等控制量，是 “人机交互” 的核心接口。 |
| `rc_modes.c/h`              | 飞行模式切换：管理遥控器的模式开关（如自稳 / 手动模式、定高 / 定点模式），实现模式的动态切换。 |
| `rc_adjustments.c/h`        | 遥控器微调：支持通过遥控器调整 PID 参数、油门曲线等，方便飞行中实时优化操控。 |

### 四、配置与状态管理

| 文件                      | 作用                                                         |
| ------------------------- | ------------------------------------------------------------ |
| `controlrate_profile.c/h` | 操控速率配置：管理不同飞行模式下的姿态角速度限制（如竞速模式允许更大的姿态角），适配不同操控习惯。 |
| `parameter_names.h`       | 参数名称定义：统一管理飞控的所有可配置参数（如 PID 值、混控模式），是参数存储与调参工具（如地面站）的交互基础。 |

### 五、安全与统计

| 文件           | 作用                                                         |
| -------------- | ------------------------------------------------------------ |
| `hardfaults.c` | 硬件故障处理：捕获 CPU 硬件故障（如内存错误、总线异常），记录故障信息并尝试安全重启，提升系统鲁棒性。 |
| `stats.c/h`    | 运行统计：记录飞控的运行时间、电池循环次数等状态信息，用于调试和维护。 |

### 六、扩展功能

| 文件                | 作用                                                         |
| ------------------- | ------------------------------------------------------------ |
| `gps_lap_timer.c/h` | GPS 圈速计时：结合 GPS 实现竞速比赛的圈速记录，是穿越机竞速场景的专用功能。 |

### 总结

`fc`文件夹是飞控的 “系统底座”，**核心作用是搭建飞控的运行框架**：从启动初始化、任务调度，到遥控器交互、参数管理，为`flight`等功能模块提供基础支撑。其中`init.c`（初始化）、`rc_controls.c`（遥控器处理）、`tasks.c`（任务管理）是飞控运行的**必用核心文件**，决定了飞控的 “启动流程” 和 “基础交互能力”。

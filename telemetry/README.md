这个`telemetry`文件夹是飞控的**遥测数据传输模块目录**，负责将飞控的状态数据（如电池电压、姿态、GPS 信息）回传至遥控器或地面站，实现 “飞控→用户” 的状态反馈，是穿越机飞行监控的核心功能层。以下是各协议模块的核心作用分类：

### 一、主流遥测协议实现

| 协议文件         | 对应遥控器 / 系统    | 核心特点（穿越机场景适配）                                   |
| ---------------- | -------------------- | ------------------------------------------------------------ |
| `smartport.c/h`  | FrSky SmartPort 协议 | 适配 FrSky 遥控器，支持实时回传电压、RSSI 等数据，是穿越机户外飞行的常用遥测方式。 |
| `crsf.h`         | Crossfire 遥测协议   | 与 Crossfire 遥控器配合，回传高带宽数据（如黑盒日志、姿态角），延迟低，适合长距离飞行监控。 |
| `ghst.c/h`       | Ghost 遥测协议       | 适配 Spektrum Ghost 遥控器，支持低延迟、多数据项回传，适配新款穿越机遥控器系统。 |
| `msp_shared.c/h` | MSP 遥测协议         | 通过 MSP 协议向地面站回传数据，是电脑端监控飞控状态的基础协议。 |

### 二、辅助功能模块

| 文件名称        | 作用                                                         |
| --------------- | ------------------------------------------------------------ |
| `telemetry.c/h` | 遥测核心逻辑：统一管理各遥测协议的启动、数据打包与发送，是飞控状态数据的 “出口”。 |
| `frsky_hub.c/h` | FrSky Hub 协议适配：支持 FrSky 遥控器的扩展遥测设备（如外部传感器），增强数据采集能力。 |

### 总结

`telemetry`文件夹是飞控的**状态反馈接口**，核心作用是将飞控的运行数据回传给用户，帮助实时监控飞行状态（如低电压告警、信号强度）。对于穿越机而言，`smartport.c`（FrSky）、`crsf.h`（Crossfire）是**高频使用模块**，其数据回传能力直接影响飞行安全性（如及时发现低电压）与调试效率（如实时查看姿态角）。

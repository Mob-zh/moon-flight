这个`rx`文件夹是飞控的**遥控器接收模块目录**，负责实现不同遥控器协议的解析，是飞控 “人机交互” 的核心入口，支撑穿越机的手动操控。以下是各协议模块的核心作用分类：

### 一、主流遥控器协议实现

| 协议文件         | 对应遥控器系统         | 核心特点（穿越机场景适配）                                   |
| ---------------- | ---------------------- | ------------------------------------------------------------ |
| `sbus.c/h`       | Futaba/SBUS 协议       | 传统数字协议，支持 16 通道，延迟中等（约 10ms），是穿越机早期常用协议。 |
| `crsf.c/h`       | Crossfire 协议         | 远距离数字协议，支持低延迟（约 2ms）、高抗干扰，适合户外长距离穿越机飞行。 |
| `expresslrs.c/h` | ELRS（ExpressLRS）协议 | 开源远距离协议，延迟极低（<1ms）、带宽高，是当前穿越机竞速 / 远航的主流选择。 |
| `spektrum.c/h`   | Spektrum 卫星协议      | 传统卫星接收协议，适配 Spektrum 遥控器，延迟中等，适合入门级穿越机。 |
| `ghst.c/h`       | Ghost 协议             | Horizon Hobby 推出的数字协议，支持低延迟、高通道数，适配 Spektrum 新款遥控器。 |

### 二、辅助功能模块

| 文件名称       | 作用                                                         |
| -------------- | ------------------------------------------------------------ |
| `rx_bind.c/h`  | 遥控器绑定功能：实现不同协议的配对流程（如 ELRS 的绑定、SBUS 的对码）。 |
| `rc_stats.c/h` | 遥控器状态统计：记录信号强度（RSSI）、丢包率等，用于监控遥控器链路质量（穿越机飞行安全的关键指标）。 |
| `pwm.c/h`      | PWM 协议实现：支持传统 PWM 遥控器（通道少、抗干扰弱，穿越机极少使用）。 |

### 总结

`rx`文件夹是飞控的**遥控器协议适配层**，核心作用是解析不同遥控器的信号，将其转换为飞控可识别的 “滚转、俯仰、油门、偏航” 控制量。对于穿越机而言，`expresslrs.c`（ELRS）、`crsf.c`（Crossfire）是**高频使用模块**，其低延迟、高抗干扰的特性，直接决定了穿越机高速飞行时的操控响应与链路稳定性。
